forge

A code-change workflow for Claude Code. Your request gets planned, the plan gets attacked, the code gets written with tests, and a panel of reviewers shoots at it before you ever see it.

Install

/plugin marketplace add alp82/forge
/plugin install forge
/setup-forge

Use

One verb. Describe the change you want in your own words.

/forge add rate limiting to the public API
/forge #482                    # or point it at a ticket
/crossfire                     # review what's already there

What happens

StageWhat it does
triageSizes the request. Too big gets carved down or sent to a mapping skill; unclear gets interviewed.
planWrites the plan to a file, not the chat.
challengeA second agent tries to break the plan before a line is written.
implementMakes the change, with tests.
test-reviewCatches tests that pass without proving anything.
crossfireIndependent reviewers hit the diff at once, each with one lens.
fixWorks the findings until the diff survives.

It can't skip the review

Prompts get forgotten under compaction; hooks don't. Forge ships six of them. If code changed and the review never ran, the session refuses to end.

It's just markdown

No agent definitions, no config language, no signal vocabulary. Every stage is a markdown file you can read in one sitting and edit with your own opinions. The plugin is a delivery mechanism, nothing more.

Works with your tracker

Point /forge at a ticket and it reads the ticket as the request, then posts the verdict back and closes it. No tracker? Nothing is missing — the contract lies dormant.

Watch it think before it types.

One request. Seven stages. Nothing hidden — this is the whole run, top to bottom.

Act one

It sizes you up first

Before planning, forge decides whether your request is even one session's work. Too big gets carved down. Vague gets questioned.

› /forge add rate limiting to the public API

triage  one session, clear enough → proceed
triage  found 4 route handlers, no existing middleware
Act two

The plan gets attacked

A second agent's only job is to break the plan — before a single line is written. Cheapest possible place to be wrong.

plan      → .forge/rate-limit/plan.md
challenge ✕ token bucket per-process breaks on 3 replicas
challenge ✕ no answer for what a 429 body says
plan      revised → redis-backed, shared bucket
Act three

Tests that actually prove something

A reviewer reads the tests looking for green that means nothing — the assertion that can't fail, the mock that tests the mock.

implement 4 files, +181 −12
tests     11 added
test-review ✕ burst test passes with the limiter deleted
fix       assertion now fails without the limiter
Act four

Crossfire

Independent reviewers hit the diff simultaneously, each seeing one thing only. They don't confer. They don't know what the others found.

crossfire 5 lenses, parallel
  ✓ correctness   clean
  ✕ security      limiter keyed on X-Forwarded-For, spoofable
  ✕ acceptance    admin routes uncovered, ticket said "public API"
  ✓ simplicity    clean
  ✓ conventions   clean

fix       2 findings → resolved
crossfire re-run ✓ clean
done      4 files, +196 −12 · plan survived challenge · 11 tests

Your agent reviews grades its own homework.

Every coding agent promises to check its work. Then the context fills up, the instruction falls out, and it ships anyway — cheerfully. Forge is built on the assumption that the model will forget, and makes forgetting impossible.

Claim
The review can't be skipped.

Not "is instructed not to be." A hook blocks the session from ending while code has changed and no review has run. Prompts are advice; hooks are physics.

Claim
The plan gets attacked before code exists.

A dedicated agent's only job is to break the plan. Wrong is cheapest at the plan stage — not after four files and a test suite.

Claim
Reviewers don't confer.

Crossfire runs the lenses in parallel and blind. One reviewer with five jobs finds one thing. Five reviewers with one job each find five.

Claim
Green tests get audited.

A reviewer hunts for the passing test that would still pass with the feature deleted. False green is worse than red.

Claim
Nothing here is a black box.

No agent definitions, no config DSL. Every stage is a markdown file under 900 words. Disagree with one? Edit it.

Claim
The chat stays out of the way.

Stages hand each other file paths, not pasted text. Runs survive compaction, and the orchestrator stops out-typing its own fleet.
